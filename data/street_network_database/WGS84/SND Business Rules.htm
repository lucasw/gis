Street Network Model Business Rules

 

Object: SNDSEG       Field: SHAPE (feature geometry)

The Street Network Model consists of Transportation Segments and Transportation Intersections which represent open and improved land travel pathways (roads, paths, and railroads) and termination or intersection of these pathways.

 

Feature Definition

Transportation Segments (SNDSEG Feature Geometry ) represent a continuous travel path terminated by path end or intersection with other travel path(s).

 

Horizontal Positional Accuracy

Street Network Database features (segments and intersections) are coincident with CGDB Survey Control layer features representing street centerlines where possible. Certain intersections and approaches to the intersections are generalized as a better representation of traffic flow patterns. Vertically coincident streets are offset to allow two-dimensional graphic representation. Where not coincident with CGDB Control layer feature, the street network feature is within its associated ROW boundary.  Where street network feature is not within a ROW, street network feature is within the pavement shown on most recent positional accurate orthophoto layer.

 

Object: SNDSEG       Field: F_INTR_ID

The identifier of the physical intersection that defines the start of a line feature.

Values automatically assigned and managed by edit system. Values based upon topological connections between Segments and Intersections.

 

 

Object: SNDSEG       Field: T_INTR_ID

The identifier of the physical intersection that defines the end of a line feature. Values automatically assigned and managed by edit system. Values based upon topological connections between Segments and Intersections.

 

 

Object: SNDSEG       Field: SND_ID

Unique identifier for segments.

For new segments, value is assigned from an Oracle Sequence.

For merged segments, value of resultant segment is that of longest source segment involved in merge operation.

For split segment, new values are assigned to both newly created segments, assigned from an Oracle Sequence.

 

Object: SNDSEG       Field: GEOBASID

GEOBASID values represent unique Seattle Department of Transportation Street Segments. Within City of Seattle, GEOBASID values assigned by SDOT for segments referenced by SDOT. For other segments within City of Seattle, but not referenced by SDOT, GEOBASID value is zero. Outside City of Seattle, value is zero.

Object: SNDSEG       Field: ST_CODE

The street code designation for all SNDSEG line segments. There are many varieties of both streets and non-streets. 

Assignment of ST_CODE is based upon interpretation of segment function, ownership and modal accessibility as classed in ST.CODE domain.

Street codes are of six types: public, private, freeway, secondary freeways, divided, and vacated with an address range.  Non-street codes are of five types: walkway, multi-purpose trail, stairs, railroad, closed street.

Sources used for street code designation include the Geofile, the Geobasys file, the Common Land Data Base, Kroll maps, King County Assessor’s maps, aerial photography, and on-site visits. All freeways and divided streets can be also regarded as public streets.

 

ST.CODE

Description

Definition

 0

Open Public Street

Existing driveable public streets legally open to traffic and currently maintained by SED.

 1

Divided Public Street

Open public streets that are represented by two parallel line segments.  Criteria to designate a street as divided can be one or more of the following: an expanded right-of-way, the street is represented by two arcs in either the Geobasys or Geofile, or a grade separation of the street.

 2

Parks Street (Unlimited Access)

Streets on Parks Department property to which the public has unlimited access.  An example of this street classification is Lake Washington Boulevard.

 4

Freeways and Freeway Ramps

A limited access multilane highway.  This classification includes on-ramps and off-ramps.

 6

Alley

A street designed for limited travel and primarily used as access to the rear of residences and businesses. In Seattle, alleys are listed as north or east of a street. SND does not have a comprehensive inventory of alleys. 

 7

Parks Street (Limited Access)

Streets on Parks Department property to which there is limited access.  For example, the streets in Magnuson Park are closed at sunset.

 8

Other Agency

Streets on property owned or maintained by a government agency other than the City of Seattle.  Examples of these streets are the federally and Metro-maintained streets of Discovery Park and the streets on the University of Washington campus.

 9

Transit Only

Limited access streets for the use of transit and emergency vehicles only.

20

Stairs

Stairs in place of a driveable street located in a right-of-way.

21

Walkway

 A path for pedestrian use only that is located in a street right-of-way.

22

Multipurpose Trail

 A path for pedestrian and non-motorized vehicle use in a street or railroad right-of-way.

40

Private Street

Existing streets on private property which are not maintained by SED.  These streets also may be located within a right-of-way and carry its address range but have been identified as privately maintained by our sources.

50

Railroad

Existing or abandoned railroad tracks within street or railroad right-of-ways.

51

Dock (Addressable Slips)

Nondriveable, private docks located in street right-of-ways which have been platted in water bodies.  For example, some houseboats located along the shores of Lake Union carry addresses to these ‘underwater streets.’

70

Inaccessible Addresses

Features that represent an address range.  These ‘streets,’ however, do not exist and access to these addresses is located elsewhere.

71

Meter Only

Features that represent an address for utility billing purposes only.

Object: SNDSEG       Field: BIKECLASS

Coding representing Seattle Department of Transportation dedicated infrastructure for bicycle usage plus recommendations for bicycle use of other shared public infrastructure. Bicycle route code values assigned by SDOT, based upon SDOT classifications.

 

Object: SNDSEG       Field: TRANCLASS

Coding representing Seattle Department of Transportation transit use code for City streets. Transit use code values assigned by SDOT, based upon SDOT classifications.

 

Object: SNDSEG       Field: SNOWICE

Coding representing Seattle Department of Transportation priority response routes for Snow and Ice removal. Snow and Ice response code values assigned by SDOT, based upon SDOT classifications.

 

Object: SNDSEG       Field: NS_CODE

A code scheme used to differentiate and classify segments representing drivable streets vs. 'other' non-drivable conditions.

Value assigned is based upon interpretation of segment characteristics as classed in NS.CODE domain.

 

Object: SNDSEG       Field: GIS_MID_XCOORD

The numeric value of the x coordinate of the calculated midpoint of the segment geometry.

Value assigned is by calculation during creation or modification of the segment geometry.

 

Object: SNDSEG       Field: GIS_MID_YCOORD

The numeric value of the y coordinate of the calculated midpoint of the segment geometry.

Value assigned is by calculation during creation or modification of the segment geometry

 

Object: SNDSEG       Field: SNDSEG_UPDATE

Date of feature creation or last modification to feature. Modification can be either to feature geometry, attribute value or combinations of both. Value assigned during save of created or modified record.
